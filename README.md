IM2026 Bureauvision: A Day in the Life of a Public Servant\n\n## Overview\nA 2–4 minute interactive web experience designed for an Australian Public Sector (APS) audience. The project simulates a professional, futuristic workplace dashboard (set in 2036) with a subtle, dystopian edge. It demonstrates the critical importance of Australia's AI Ethics Principles—specifically human autonomy, fairness, and accountability.\n\n## Project Structure\nThis project has been modularised for clean deployment and ease of development:\n\n*   `index.html` - The core structure, UI framework, and media hooks.\n*   `style.css` - Minimalist, corporate styling mirroring modern APS enterprise dashboards.\n*   `app.js` - Branching narrative logic, DOM manipulation, and audio/video controllers.\n*   `bg.mp3` - Background ambient audio loop (needs to be provided/uploaded).\n*   `images/` - Directory for any static image assets (e.g., logos, placeholders).\n\n## Media Assets\nThe experience requires media files to fully function. Placeholders are referenced in the HTML.\n*   **Root:** `bg.mp3` (replaces the ambient hum from previous concepts).\n*   **Subdirectories (to be created):**\n    *   `audio/ui-click.mp3`\n    *   `audio/ethical-dilemma-drone.mp3`\n    *   `audio/system-halt.wav`\n    *   `video/data-flow-loop.mp4`\n    *   `video/glitch-transition.mp4`\n\n## The Narrative\n1.  **Phase 1 - Illusion of Efficiency:** The user 'auto-resolves' tasks, experiencing a false sense of security and agency.\n2.  **Phase 2 - The Halt:** A critical ethical dilemma arises. The user attempts to intervene, but the system aggressively rejects human input (simulated via auto-deleting text).\n3.  **Phase 3 - The Message:** A stark reminder that AI should augment judgement, not replace it.\n\n## Deployment\nSimply host the root directory on GitHub Pages or any standard static web server. No backend or external frameworks (React/Vue/etc.) are required. Pure HTML, CSS, and vanilla JS.
